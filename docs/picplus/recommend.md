# 图床推荐

该文专为 [咕咚云图](https://www.coolapk.com/apk/name.gudong.pic) 写作，意在为大家提供一个图床选择的指南。咕咚云图是一个图床上传 APP，可帮你上传手机图片到图床，并生成 markdown 链接。



在国内，可选的图床服务有不少，主流的有阿里云、七牛云等，小众的图床服务也很多，后面大都依赖大厂提供的云存储。



咕咚云图支持七牛云、阿里云等主流图床，同时还支持配置 GitHub、码云为个人图床。



下面是根据个人使用经验，对不同图床进行打分，满分5星。

## 牛图


推荐指数：★★★



免费图床，速度一般，牛图不提供删除、管理图片机制，所以**只能传，不能删**，PicPlus 中对牛图图片的删除**仅是删除本地上传记录**，如下是牛图官方对于图片的删除操作的申明：



> 没有服务协议,已上传的图片均不可删除! 对的,你只能上传,不能删除,图片会永久存在,所以请想清楚在上传!
>



**PicPlus 建议：**



> 牛图依旧是一个小众的图床服务，不要过于依赖。
>



更多牛图相关的信息，请移步[牛图网 - 免费图片空间](https://niupic.com/)





## 搜狗图床


推荐指数：★★★



使用搜狗搜索（识图）的接口进行上传，上传下载速度极快，不过只能存储 24 小时，过期会被搜狗删除，搜狗图床适合一些临时的存储展示需求。





## 码云


推荐指数：★★★★



码云是国内的代码、文件托管平台，任何用户注册即可使用，上传下载速度快，而且上传的图片都在自己的账号下面，可以更加灵活的管理删除。不过码云也有一些限制。



+ 容量限制



> 规定每个仓库的大小上限为1个G，单个用户总容量为5个G。
>
> [from gitee terms](https://gitee.com/terms)
>



+ 图片大小限制



> 单个图片大小超过 1M 需要登录才可访问。
>





## Github


推荐指数：★★★★★



GitHub 是全球知名的代码、文件存储平台，跟国内的码云一样，只需要注册账号、建立仓库即可使用，不过 GitHub 由于服务器在国外的原因，所以访问速度欠缺，你可以选择启用 CDN 加速，以便让 GitHub 上的图片有更好访问速度。



另外，GitHub 的存储空间理论上不受限制，可随意使用，不过单仓库如果大小超过 1G，需要联系 GitHub 才可以继续使用。



> GitHub doesn't have any set disk quotas. We try to provide abundant storage for all Git repositories, within reason. Keeping repositories small ensures that our servers are fast and downloads are quick for our users.If your repository exceeds 1GB, you might receive a polite email from GitHub Support requesting that you reduce the size of the repository to bring it back down.
>
> from [https://help.github.com/en/github/managing-large-files/what-is-my-disk-quota](https://help.github.com/en/github/managing-large-files/what-is-my-disk-quota)
>



## SM.MS


推荐指数：★★★



这是小众图床，但是已经存活了好几年，还算比较稳定，但是速度稍慢，不过相比牛图等，它支持账号系统，你上传的图片可以继续管理。



另外 SM.MS 图床的配置很简单，只需要一个 token 就OK，好评。

## 总结


阿里云、腾讯云都是按量收费，对个人用户，一般的存储量花费不了多少钱，但图片下载、访问也会计费，如果你的图片用在流量较大的网站，因为用户访问过多可能会带来不少的下载费用，请知悉，具体请仔细阅读相关云服务的计费指南。



下面从两个角度简单总结一下：



### 按场景


+ 图片对外可访问，如博客用图，建议使用 **Github。**



> 码云也可以，速度也优于 GitHub，但是图片大小有限制，不能超过 1M，如果使用，可配合 PicPlus 的压缩使用。
>



+ 私有笔记或者只是小范围才可以访问，推荐使用 **阿里云，**快速稳定，值得拥有。



### 按稳定性


+ 如果你对**稳定性、访问速度**要求较高，则**推荐使用阿里云**，阿里云的费用在可接受的范围内，具体看下文。
+ 如果只是**轻度图床用户**、不想破费的同学推荐使用 **GitHub、码云**或者 SM.MS。



以上仅为自己的一些建议指南，不一定完全正确，各位参考就好。
